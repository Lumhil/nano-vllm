# 为什么叫 nano-vLLM
它是对著名项目 vLLM 的简化复刻。vLLM 有几十万行代码，而 nano-vLLM 只有约 1200 行 Python 代码，所以叫 “nano”（纳米级）。虽然精简，但它实现了 vLLM 的几个核心技术：

技术	一句话解释
Prefix Caching（前缀缓存）	相同的输入前缀只算一次，后续复用
Tensor Parallelism（张量并行）	把模型切分到多个 GPU 上并行计算
CUDA Graph（CUDA 图）	把 GPU 操作录制成”录像带”，减少启动开销
Torch Compile（即时编译）	让 PyTorch 自动优化计算图
Chunked Prefill（分块预填充）	把长输入切成小块处理，避免卡顿
